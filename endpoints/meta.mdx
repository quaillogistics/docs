---
title: Metadata
api: "GET /v1/meta"
description: Database statistics, data freshness, and feature horizons
---

Returns database statistics, the timestamp of the latest ETL run, and the horizons consumers need for [risk-history](/endpoints/risk-history) and [point-in-time](/endpoints/get-carrier#point-in-time-as-of) queries.

<RequestExample>
```bash cURL
curl https://api.dotlookup.dev/v1/meta
```
</RequestExample>

<ResponseExample>
```json 200
{

  "active_carriers": 400489,
  "carriers_with_safety": 305298,
  "carriers_with_insurance": 425587,
  "last_etl_run": {
    "id": 462,
    "dataset": "export",
    "started_at": "2026-05-19T08:00:00Z",
    "finished_at": "2026-05-19T08:18:36Z",
    "rows_processed": 1679124,
    "rows_errored": 0,
    "status": "success",

  "data_freshness": "2026-05-19T08:18:36Z",

  "risk_history_rows": 1679124,

}
```
</ResponseExample>

## Fields

| Field | Description |
|---|---|
| `total_carriers`, `active_carriers`, `carriers_with_safety`, `carriers_with_insurance` | Row counts in the corresponding tables. `active_carriers` filters `carrier_status = 'ACTIVE'`. |
| `last_etl_run` | Metadata for the most recently completed ETL step (typically `export`, the final step). |
| `data_freshness` | `finished_at` of `last_etl_run`. Use this as the "as of" date for the live response. |
| `risk_history_horizon` | Earliest `computed_at` in `carrier_risk_history`. Floor for the [risk-history](/endpoints/risk-history) endpoint. Absent when the table is empty (pre-first-ETL state). |
| `risk_history_rows` | Total rows in `carrier_risk_history`. `0` is a meaningful "no history yet" signal. |
| `etl_changes_horizon` | Earliest `batch_ts` in `etl_changes`. Floor for [point-in-time `?as_of=` queries](/endpoints/get-carrier#point-in-time-as-of). Retention is 52 weeks. Absent when the change feed is empty. |
